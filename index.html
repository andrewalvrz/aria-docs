03 — ESP32 micro-ROS State Machine
STATE ESP32 Firmware Lifecycle
Internal state of the ESP32 micro-ROS node from boot to active publishing. The 15-second delay before /cmd_vel responds is caused by the WAITING_FOR_AGENT → AGENT_CONNECTED transition.
stateDiagram-v2 [*] --> BOOT : power on BOOT --> INIT_HARDWARE : setup motors\nIMU · encoders · ToF INIT_HARDWARE --> WAITING_FOR_AGENT : hardware ready\nstart micro-ROS transport WAITING_FOR_AGENT --> WAITING_FOR_AGENT : ping agent\nevery 500ms WAITING_FOR_AGENT --> AGENT_CONNECTED : agent responds\nsession created AGENT_CONNECTED --> CREATING_ENTITIES : create publishers\n+ subscriber CREATING_ENTITIES --> AGENT_AVAILABLE : all entities created\n/odom /imu /tof /cmd_vel AGENT_AVAILABLE --> SPINNING : enter main loop SPINNING --> SPINNING : publish /odom @ 7Hz\npublish /imu/data @ 7Hz\npublish /tof/left /tof/right\nreceive /cmd_vel → drive motors SPINNING --> WAITING_FOR_AGENT : agent disconnect\nor serial error note right of WAITING_FOR_AGENT This is why cmd_vel has ~15s delay on first connect — agent must re-establish end note

06 — RTAB-Map SLAM State Machine
STATE SLAM Mapping Lifecycle
How RTAB-Map builds and maintains the map using the OAK-D Lite RGB-D camera and EKF odometry. Loop closure is key for correcting drift.
stateDiagram-v2 [*] --> INITIALIZING : rtabmap_ros node start\nwaiting for camera + odom INITIALIZING --> MAPPING : first RGB-D frame\n+ odometry received MAPPING --> MAPPING : add node to graph\nevery 1s or 0.5m moved\nextract visual features MAPPING --> LOOP_CLOSURE_CHECK : new node added\nsearch for similar\npast nodes LOOP_CLOSURE_CHECK --> MAPPING : no match found\ncontinue mapping LOOP_CLOSURE_CHECK --> LOOP_CLOSURE_DETECTED : visual similarity\nabove threshold LOOP_CLOSURE_DETECTED : Loop Closure Detected LOOP_CLOSURE_DETECTED --> GRAPH_OPTIMIZATION : optimize pose graph\ncorrect accumulated drift GRAPH_OPTIMIZATION --> MAPPING : updated /map published\nto Nav2 + SENTINEL MAPPING --> LOCALIZATION_ONLY : map saved\nswitch to localization mode LOCALIZATION_ONLY --> LOCALIZATION_ONLY : localize against\nsaved map\nno new nodes added note right of LOOP_CLOSURE_DETECTED This is what fixes drift on the odometry map in SENTINEL over long runs end note
